# Contributing to StackCraft

First off, thank you for considering contributing to StackCraft! It's community-driven effort to make system design knowledge accessible to everyone.

## How to Contribute

### 1. Suggesting Improvements
If you find a typo, an outdated architectural pattern, or a missing case study, please open an **Issue** on GitHub. Describe the problem and suggest a potential fix.

### 2. Submitting a Pull Request
If you'd like to contribute code or content directly:

1. **Fork** the repository.
2. Create a **new branch** for your feature or fix:
   `git checkout -b feature/my-awesome-contribution`
3. Make your changes and ensure the site builds locally:
   `npm run build`
4. Commit your changes with a clear message.
5. **Push** to your fork and submit a **Pull Request**.

## Contribution Guidelines

### Content Standards
- **Clarity First**: Use simple language. Avoid overly complex jargon without explaining it.
- **Visuals**: Whenever possible, use Mermaid diagrams to illustrate architectural flows.
- **Citations**: If you're referencing a specific paper or blog post, please provide a link.

### Review Process
All contributions will be reviewed by the maintainer. We look for:
- Accuracy of the technical information.
- Adherence to the site's formatting and style.
- Quality of the explanations.

We look forward to your contributions!
